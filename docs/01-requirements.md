# 01. 요구사항 (Requirements)

## 프로젝트 개요

| 항목 | 내용 |
|---|---|
| 앱 한 줄 정의 | 팀이 함께 업무를 나눠 맡고 진행 상태를 보는 **칸반 보드형 할 일 앱** |
| 주된 목적 | 개인 학습/연습 |
| 주 사용 디바이스 | **반응형** (데스크톱 + 모바일 둘 다) |
| 기술 스택 | **Next.js (App Router) + TypeScript**, Vercel 배포 (Phase 6 확정) |
| 데이터 저장 방식 | **Supabase (Postgres + Realtime)** — 원격 DB (Phase 6 확정) |
| 시간 예산 | **며칠** (실시간·백엔드 포함으로 하루 → 확대) |

## 협업 모델 (MVP로 승격됨)

- **실제 멀티유저 실시간 협업**: Supabase로 하나의 보드를 여러 사람이 동시에 보고 편집한다.
  카드 생성/수정/이동/삭제가 **Postgres 변경 구독**으로 모두의 화면에 실시간 반영된다.
- **Presence(접속자 상태)**: 커서 위치 + "누가 어떤 카드를 편집 중"인지 실시간 표시.
- **인증 없는 공유 보드**: 로그인 없이, 접속 시 팀원 목록에서 "나"를 선택하는 방식.
  members 테이블만 사용. (로그인/계정 기반 인증은 **[추후 개선](#추후-개선-사항future)**)

## 기능 요구사항

### 데이터 모델 (개념 수준 — 확정 스키마는 Phase 6)

- **Board(보드)**: 단일 보드 하나. 3개 고정 컬럼을 가짐.
- **Column(컬럼)**: 고정 3단계 — `To Do` / `Doing` / `Done`. (추가·삭제·이름변경 없음)
- **Card(카드)**:
  - 제목 (필수)
  - 설명/메모 (선택, 여러 줄)
  - 담당자 (팀원 목록에서 지정, 미배정 허용)
  - 마감일 (선택)
  - 우선순위 (예: 낮음 / 보통 / 높음)
- **Member(팀원)**: 이름(+식별 색/이니셜). 팀원 목록에서 추가·삭제.

### 핵심 기능

1. **카드 CRUD** — 카드 생성 / 조회 / 수정 / 삭제.
2. **드래그 앤 드롭** — 네이티브 HTML5 Drag & Drop 사용.
   - 컬럼 간 카드 이동 (To Do ↔ Doing ↔ Done).
   - **같은 컬럼 내 순서 변경**.
   - **모바일 대체 수단**: 드래그 대신 카드 편집에서 상태(컬럼)를 바꿔 이동.
3. **담당자 배정** — 팀원 목록 관리 + 카드에 담당자 지정/변경.
4. **카드 상세 필드 편집** — 설명·마감일·우선순위 편집.
5. **진행 상태 가시화** — 컬럼별 카드 수, 담당자/우선순위/마감일이 카드에 시각적으로 드러남.
6. **데이터 영속** — Supabase에 저장, 새로고침·재접속해도 보드 상태 유지.
7. **실시간 동기화** — 다른 사용자의 변경이 즉시 반영(구독) + 커서/편집중 표시(Presence).
8. **테마** — 파스텔 4종 + 다크, 기본 A(Periwinkle & Mint). 선택은 클라이언트에 저장.

## 비기능 요구사항

- **반응형**: 데스크톱(3컬럼 가로 배치) ~ 모바일(세로 스택 또는 컬럼 스와이프)까지 사용 가능.
- **성능**: 카드 수십~수백 개에서 드래그가 버벅이지 않을 것.
- **접근성**: 키보드 조작 및 명확한 대비/포커스 표시 지향(세부는 디자인 단계).
- **한글 입력**: IME 조합 중 입력 손실/중복 없이 카드 제목·메모 작성 가능.
- **데이터 안정성**: 저장 실패나 손상 시에도 앱이 깨지지 않고 빈 보드로 복구.

## 범위 제외 (Out of Scope — MVP에서 제외)

- **계정 로그인 / 권한 관리** (인증 없는 공유 보드로 진행)
- 다중 보드·프로젝트 전환
- 커스텀 컬럼(추가·삭제·이름변경)
- 라벨/태그, 체크리스트(하위 작업), 첨부파일, 코멘트
- 알림/리마인더, 활동 로그, 검색·필터 고급 기능

## 완료 기준 (Definition of Done)

- [ ] To Do / Doing / Done 3컬럼 보드가 반응형으로 표시된다.
- [ ] 카드를 만들고(제목·메모·담당자·마감일·우선순위) 수정·삭제할 수 있다.
- [ ] 팀원을 추가/삭제하고 카드에 담당자로 지정할 수 있다.
- [ ] 드래그 앤 드롭으로 컬럼 간 이동 + 같은 컬럼 내 순서 변경이 된다. (모바일은 편집으로 상태 변경)
- [ ] Supabase에 저장되어 새로고침·재접속 후에도 보드 상태가 유지된다.
- [ ] 두 브라우저에서 동시에 열면 변경이 실시간 반영되고, 커서/편집중이 표시된다.
- [ ] 5개 테마(파스텔 4 + 다크)를 전환할 수 있고 기본은 A다.
- [ ] 데스크톱과 모바일 화면 모두에서 사용 가능하다.
