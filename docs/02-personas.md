# 02. 페르소나 (Personas)

주 페르소나 1명(PM) + 대조 페르소나 2명(꼼꼼한 실무자 / 대충 쓰는 실무자).
세 사람의 긴장 축에서 설계 원칙을 도출한다.

공통 맥락: **5명 규모** 팀이 **하루에 여러 번** 보드를 열어 상태를 업데이트한다.
세 사람 모두 기존 도구의 불편함으로 ①**누가 뭘 맡았는지 안 보임** ②**너무 무겁고 복잡** ③**단계 이동이 번거로움** 을 공유한다.

---

## 🎯 주 페르소나 — 지훈 (PM / 팀 리더, 관리자 역할 겸함)

| 항목 | 내용 |
|---|---|
| 프로필 | 32세, **5명 규모** 프로젝트를 이끄는 PM. **업무 중엔 데스크톱** 주 사용, **쉬는 시간·퇴근 후엔 모바일**로 상태 확인. |
| 목표 | 일을 팀원에게 **나눠 맡기고**, 지금 무엇이 어디까지 왔는지 **한눈에** 파악. 동시에 **마감·우선순위를 촘촘히 관리**해 리스크를 미리 포착(관리자 역할 겸함). |
| 불만 | 담당자가 흩어져 안 보이고(①), 기능이 많아 세팅에 시간 낭비(②), 급한 일이 시각적으로 안 도드라짐. |
| 행동 패턴 | 업무 중 데스크톱으로 새 카드 추가·담당자 지정·우선순위/마감 조정. 마감 임박·High 우선순위 카드부터 점검. 퇴근 후엔 폰으로 진행 상황만 훑음. |
| 핵심 니즈 | "누가/무엇을/언제까지/어느 단계"가 카드 한 장에 즉시 읽히고, 마감·우선순위가 **강하게 시각화**되며, **데스크톱↔모바일 모두** 편해야 함. |

## 🔍 대조 페르소나 A — 수민 (꼼꼼한 실무자)

| 항목 | 내용 |
|---|---|
| 프로필 | 27세, 배정받은 일을 처리하는 실무자. **업무 중엔 데스크톱** 주 사용, 이동 중엔 **모바일로 확인**. |
| 목표 | 내가 맡은 카드를 정확히 챙기고, 메모·마감을 꼼꼼히 관리하며 단계가 바뀌면 바로 반영. |
| 불만 | 단계 옮기는 절차가 번거롭고(③), 카드 정보가 부족하면 불안함. |
| 행동 패턴 | 데스크톱에서 내 카드의 메모·마감을 세세히 편집, 완료 시 Done으로 이동. 모바일에선 확인 위주. |
| 핵심 니즈 | 담당자 기준 **내 것 식별**이 쉽고, 상세 편집이 **정확**해야 함. |

## 🏃 대조 페르소나 B — 태오 (대충 쓰는 실무자)

| 항목 | 내용 |
|---|---|
| 프로필 | 25세, 최소한만 관여하는 실무자. **주로 모바일**, 한 손으로 슥 본다. |
| 목표 | 내 카드만 빠르게 확인하고, 끝내면 **곧바로 다음 단계로 이동**. 그 이상은 안 함. |
| 불만 | 조작이 조금만 번거로워도 안 씀. 필드가 많으면 부담. |
| 행동 패턴 | 폰으로 "내 카드" 위치만 훑고 → 완료되면 Done으로 넘김. 메모·마감은 거의 안 건드림. |
| 핵심 니즈 | 이동이 **한 손 동작**으로 끝나고, 필수 정보만 보이면 됨. 빈 값이어도 앱이 자연스러워야 함. |

---

## 긴장 축 비교

| 축 | 지훈 (PM) | 수민 (꼼꼼 실무자) | 태오 (대충 실무자) |
|---|---|---|---|
| 주 디바이스 | 데스크톱(업무)+모바일(퇴근후) | 데스크톱(업무)+모바일(확인) | **모바일 중심** |
| 관심 대상 | 보드 전체 + 마감·우선순위 | 내 카드(정밀) | **내 카드만(최소)** |
| 주 동작 | 배정·구성·리스크 점검 | 상세 편집 | **단계 이동** |
| 정보 요구 | 요약 + 정밀 둘 다 | 풍부·정밀 | 최소·빠름 |
| 속도 vs 정밀 | 균형 | **정밀** | **속도** |
| 관여도 | 높음 | 높음 | **낮음** |

## 설계 긴장점 (여기서 설계 원칙이 나온다)

- **T1. 한눈 요약 ↔ 정밀 편집**: 카드 겉면은 담당자·마감·우선순위를 요약해 보여주고, 상세 편집은 별도 뷰로 분리해 화면이 복잡해지지 않게.
- **T2. 데스크톱 3컬럼 ↔ 모바일 한 손 조작**: 세 명 모두 모바일을 쓰며, 특히 태오는 모바일 중심 → 모바일에서 세로 스택 + 쉬운 단계 이동 수단 필수(드래그가 어려운 상황 대비).
- **T3. 내 카드 식별성**: 담당자 색/이니셜로 "내 것"이 즉시 눈에 띄어야 함(수민·태오).
- **T4. 긴급도 시각화**: 임박 마감·High 우선순위가 카드에서 강하게 도드라져야 함(지훈).
- **T5. 간결함 유지 + 빈 값 허용**: 태오처럼 최소만 입력해도 자연스럽고, 지훈·수민처럼 정밀하게 채워도 감당되게 — 기능은 늘리되 화면은 단순하게.
