# 03. 시나리오 (Scenarios)

페르소나별 실제 사용 장면 8개. 각 시나리오의 인사이트를 모아 설계 원칙 P1~P7로 집약한다.
등장인물: **지훈**(PM), **수민**(꼼꼼 실무자), **태오**(대충 실무자).

---

## S1. 킥오프 회의 — 할 일 쏟아내기 (지훈)

- **Context**: 프로젝트 시작. 회의 중 해야 할 일들이 우르르 나온다.
- **Flow**: 지훈이 데스크톱에서 To Do에 카드를 **연속으로 빠르게** 여러 개 생성(제목만). 나중에 담당자·마감은 채운다.
- **인사이트**: 카드 생성은 **제목만으로 즉시** 만들어지고, 만든 뒤 바로 다음 카드를 이어 만들 수 있어야 한다(연속 입력). → **P1, P5**

## S2. 업무 중 새 요청 — 즉시 배정 (지훈)

- **Context**: 작업 도중 새 요청이 들어온다.
- **Flow**: 지훈이 카드 한 개를 만들고 **곧바로 담당자 지정 + 우선순위**를 걸어 담당자에게 넘긴다.
- **인사이트**: 생성 흐름 안에서 담당자·우선순위 지정이 **한 화면에서** 끝나야 한다. → **P1, P3**

## S3. 아침 계획 세우기 (수민)

- **Context**: 하루 시작. 수민이 오늘 할 일을 정리한다.
- **Flow**: 내게 배정된 카드를 훑고, 오늘 할 것을 To Do→Doing으로 옮기고 순서를 정리.
- **인사이트**: "내 카드"가 담당자 색/이니셜로 **한눈에 식별**되고, 순서 변경이 자연스러워야 한다. → **P2, P4**

## S4. PM 리스크 점검 — 어디서 막혔나 (지훈)

- **Context**: 지훈이 마감 임박·High 우선순위 일이 어디서 정체됐는지 확인.
- **Flow**: 보드 전체에서 **급한 카드가 도드라져** 바로 눈에 들어오고, 막힌 카드를 클릭해 상태 조정.
- **인사이트**: 임박 마감·High 우선순위는 **강한 시각 신호**(색/뱃지)로 표시. Doing에 카드가 몰리면 과부하가 드러나야. → **P6, P7**

## S5. 퇴근 후 모바일 확인 (지훈)

- **Context**: 퇴근 후 소파에서 폰으로 "오늘 얼마나 진행됐나" 확인.
- **Flow**: 모바일 세로 화면에서 컬럼별 카드 수/Done 진척을 훑는다. 편집은 거의 안 함.
- **인사이트**: 모바일에서 **읽기 경험**이 좋아야 하고, 컬럼별 카운트가 보여야 한다. → **P2, P6**

## S6. 태오의 모바일 확인 — 내게 온 새 카드 (태오)

- **Context**: 태오가 폰으로 슥 열어 본다.
- **Flow**: 태오가 가장 자주 하는 단 하나의 동작 — **내게 배정된 새 카드 확인**. 필요하면 한 손으로 Done까지 이동.
- **인사이트**: 담당자 필터/식별로 "내 카드"에 빠르게 도달하고, **모바일 단계 이동이 한 손 동작**(드래그가 어려우면 대안 조작)으로 끝나야. → **P2, P4**

## S7. 수민의 상세 정리 (수민)

- **Context**: 수민이 데스크톱에서 자기 카드를 공들여 정리.
- **Flow**: 카드를 열어 **메모·마감**을 상세히 적고, 완료 전 내용을 점검.
- **인사이트**: 상세 편집은 별도 뷰(모달/패널)에서 **정밀하게**. 카드 겉면은 요약만. → **P1, P5**

## S8. 수민의 순서 재배열 (수민)

- **Context**: 우선순위가 바뀌어 오늘 작업 순서를 다시 짠다.
- **Flow**: 같은 컬럼(Doing) 안에서 카드를 **드래그로 위아래 재정렬**.
- **인사이트**: 컬럼 간 이동뿐 아니라 **같은 컬럼 내 순서 변경**이 매끄러워야. → **P4**

---

## 엣지 상황 (시나리오에 녹인 것)

| 엣지 | 어떤 문제 | 설계로 대응 |
|---|---|---|
| 담당자 미배정 카드가 쌓임 | "누구 것도 아닌 일" 발생 | 미배정 카드를 시각적으로 구분(회색/점선 등) → **P3** |
| 마감 지난 카드 방치 | 놓친 일이 묻힘 | 지난 마감은 경고색으로 강조 → **P6** |
| Doing에 카드가 몰림(과부하) | 병목/과부하 | 컬럼별 카운트 표시로 과부하가 드러남 → **P7** |

## 설계 원칙 (Design Principles)

| # | 원칙 | 근거 시나리오 |
|---|---|---|
| **P1** | **생성은 마찰 없이** — 제목만으로 즉시 카드 생성, 상세는 나중에/별도 뷰에서. | S1, S2, S7 |
| **P2** | **모바일 읽기·이동 최적화** — 세로 스택, 한 손 조작, 컬럼 카운트 노출. | S3, S5, S6 |
| **P3** | **담당자 가시성** — 담당자 색/이니셜로 "내 것" 식별, 미배정은 명확히 구분. | S2, S3, S6 |
| **P4** | **자연스러운 이동** — 컬럼 간 이동 + 같은 컬럼 내 순서 변경, 모바일 대안 조작 제공. | S3, S6, S8 |
| **P5** | **요약 겉면 / 정밀 상세** — 카드는 요약, 편집은 별도 뷰로 분리. | S1, S7 |
| **P6** | **긴급도 시각화** — 임박·지난 마감, High 우선순위를 강한 신호로 표시. | S4, S5, 엣지 |
| **P7** | **과부하 가시화** — 컬럼별 카드 수로 병목을 드러냄. | S4, 엣지 |
